(constant) API_BASE_URL

This Settings are related to the pokeapi.co. - API_BASE_URL: Base URL for the pokeapi. - SET_LIMIT = 0 -> API Base Value of 20 - OFFSET is used for calculatedOffset() and increases with countPokemonLoaded (global). It's not necessary to set this, only if you want to skip some pokemon. - dreamworldSprites exist only for the first 649 Pokemon; i use this as a const only for readability of code.

calculateLastSetsMaxIndex()

The base length of a set is given in script.js => const SET_LIMIT. But when every pokemon is loaded it could be, that the last set contains less than SET_LIMITs items. To prevent bugs when go to next/prev-Pokemon in Detail-View it's important to analyse the correct length of the set.
